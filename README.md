# Gait Capture App

Локальное desktop-приложение для регистрации походки по данным RGBD-камеры,
сохранения исследовательских сессий и визуализации количественных метрик.

## Что делает приложение

- подключает `Intel RealSense D455/D455f` через `pyrealsense2`;
- записывает сырой поток в `*.bag`;
- оценивает 3D-ключевые точки по схеме `RGB -> 2D pose -> depth fusion -> 3D`;
- сохраняет метаданные сеанса, временные ряды позы и итоговые метрики;
- отображает предпросмотр RGB/depth и графики результатов;
- поддерживает `demo`-режим без камеры для проверки интерфейса.

## Рекомендуемое окружение

Для реальной работы с камерой рекомендуется Python `3.10-3.12`, так как
совместимость бинарных пакетов `pyrealsense2` и `mediapipe` обычно лучше в
этом диапазоне, чем на самых новых версиях Python.

## Установка

```bash
python3 -m venv .venv
source .venv/bin/activate
pip install -e '.[camera]'
```

Если пока нужен только интерфейс и просмотр сохранённых сеансов:

```bash
pip install -e .
```

## запуск на mac os с отсутствием mac os wheel y pyrealsense2

На macOS стандартная команда `pip install -e '.[camera]'` может не сработать,
потому что для `pyrealsense2` обычно нет подходящего `macOS` wheel. На этом
Mac приложение и офлайн-обработка `*.bag` запускались так:

```bash
/opt/homebrew/bin/python3.11 -m venv .venv311
source .venv311/bin/activate
pip install -e . pyrealsense2-macosx==2.56.5 mediapipe==0.10.14
export MPLCONFIGDIR="$PWD/.mplconfig"
python -m gait_capture_app.offline archive
```

Что здесь важно:

- используется `Python 3.11`;
- вместо `pyrealsense2` ставится `pyrealsense2-macosx`;
- для офлайн-обработки архивов используется CLI `gait_capture_app.offline`;
- `MPLCONFIGDIR` указывает `matplotlib` на локальную writable-папку.

## Запуск

```bash
python3 main.py
```

Или:

```bash
gait-capture
```

## Структура данных

Каждый сеанс сохраняется в `data/sessions/<session_id>/`:

- `metadata.json` — сведения об испытуемом и протоколе;
- `landmarks.csv` — 3D-координаты ключевых точек по кадрам;
- `metrics.json` — итоговые показатели походки;
- `recording.bag` — сырой архив RGBD-потока RealSense, если запись включена.

## Базовые метрики

Текущая версия считает:

- длительность прохода;
- оценку средней скорости;
- частоту шагов;
- среднюю длину шага;
- среднюю длину цикла;
- временные ряды углов в коленных и тазобедренных суставах.

## Ограничения текущей версии

- расчёт событий шага пока ориентирован на один испытуемый в кадре;
- точность фаз шага зависит от качества depth-карты и видимости стоп;
- для клинического использования нужна отдельная валидация на эталонной системе.
